1. UDFV GUI

1.1. UDFV GUI について

UDFV GUIは、イメージファイル・XMLファイルの相互変換、UDFファイルシステムのエラー検証といった、UDFV toolsで用意されている機能をGUI上で行うためのツールです。

またUDFV toolsとは異なり、Windows PCに搭載されたドライブから、直接イメージを読み込むことができます。そのため、メディアから一旦イメージデータを抽出する手間がかかりません。

1.2. 構成

UDFV GUIは、実際に操作を行うGUIクライアントと、ドライブからディスクイメージを読み取るためのATAPIServer.exeから構成されます。ATAPIServer.exeは、ATAPIServerディレクトリに格納されています。

以下、特に明記がない限り、このGUIクライアントのことをUDFV GUIと呼びます。

1.3. 動作環境

1.3.1. UDFV GUI

UDFVライブラリが動作する環境が必要です。詳細は、UDFVライブラリの動作環境ドキュメントを参照してください。

1.3.2. ATAPIServer.exe

DVD-R/RWドライブを搭載したWindows PC上で動作します。Windows XP(Service Pack 2)、Windows 2000(Service Pack 4)でそれぞれ動作を確認しています。

DVD-R/RWドライブは、ATAPIコマンドで動作する必要があります。実際の検証では、IDE接続とFirewire接続の環境で動作を確認しています。また、検証に使用したドライブはPioneer製 DVR-103、DVR-104です。

1.4. インストール

1.4.1. UDFV GUI

UDFVライブラリがインストールされていればそのまま実行可能です。UDFVライブラリのインストールについては、UDVライブラリの動作環境ドキュメントを参照してください。

1.4.2. ATAPIServer.exe

Windows PC上の任意の場所にコピーしてください。メディアから直接データを読み込む必要がない場合、この作業は特に必要ありません。

1.5. UDFV GUI の実行

1.5.1. UDFV GUI

UDFVライブラリへのクラスパスの指定と、XML パーサーライブラリへのクラスパスの指定が必要となります。

FreeBSD や Linux などの PC-UNIX 環境でコマンドラインオプションとして指定するときは、次のようになります。

例）カレントディレクトリに udfv.jar ファイルがあり、Xerces2 Java Parser 2.6.2 Release が
    カレントディレクトリに展開されている場合。

> java -Xmx128m -cp ./udfv.jar:./xerces_2_6_2/xercesImpl.jar:./xerces_2_6_2/xml-apis.jar com.udfv.gui.UDFVGui

Windows 2000 や Windows XP ではクラスパスの区切り文字が異なり、コロンではなく、セミコロンで指定する必要があります。

例）Windows 2000, XP の場合。

C:\UDFV> java -Xmx128m -cp .\udfv.jar;.\xerces_2_6_2\xercesImpl.jar;.\xerces_2_6_2\xml-apis.jar com.udfv.gui.UDFVGui

コマンドラインオプションで指定することを省きたい場合は、クラスパスを環境変数に設定する、もしくは JRE の拡張用ディレクトリに各ライブラリを複写する、などのことを行っておきます。

参考）Windows XP に JRE をインストールした場合の拡張用ディレクトリ。

C:\Program Files\Java\jre1.4.2_08\lib\ext\

このディレクトリはインストール時の設定によって変化します。詳細は Sun の Java Web ページをご覧ください。

1.5.2. ATAPIServer.exe

特別な準備は必要ありません。エクスプローラ上でダブルクリックして起動してください。

起動すると、UDFV GUIからの受け付け待ち状態になります。

1.6. メニュー

1.6.1. File メニュー

1.6.1.1. Read from File

ハードディスク上に保存されているイメージファイルを読み込みます。選択するとファイルダイアログが開きます。読み込むイメージファイルを選択してください。

ツールバー上のRead Image Fileをクリックしても、同様に読み込むことができます。

1.6.1.2. Read from XML

UDFVライブラリによって生成される、XMLドキュメントファイルを読み込みます。選択するとファイルダイアログが開きます。読み込むXMLファイルを選択してください。

ツールバー上のRead XML Fileをクリックしても、同様に読み込むことができます。

1.6.1.3. Read From Disc

Windows PC上のドライブに挿入されているメディアから、直接イメージデータを読み込みます。Windows PCでは、ATAPIServer.exeが起動している必要があります。

選択すると、Read Discダイアログが開きます。

  • IP Address: ATAPIServer.exeが動作しているWindows PCのIPアドレスを指定します。
  • Drive: メディアが挿入されているドライブのドライブ文字を指定します。

ツールバー上のRead from Discをクリックしても、同様に読み込むことができます。

1.6.1.4. Write File

イメージデータ、XML、またはメディアから読み込んだイメージデータを、イメージファイルとして書き出します。選択すると保存ダイアログが開きます。ファイル名を指定して保存してください。

1.6.1.5. Write XML

イメージデータ、XML、またはメディアから読み込んだイメージデータを、XMLドキュメントファイルとして書き出します。選択すると保存ダイアログが開きます。ファイル名を指定して保存してください。

1.6.1.6. Write Disc

イメージデータ、XML、またはメディアから読み込んだイメージデータを、メディアに書き出します。1.6.1.3.と同様に、書き出す対象のIPアドレスとドライブ文字を指定してください。

但し現状では試作段階のため、十分な書き込み速度が出ない場合があります。

1.6.1.7. Exit

UDFV GUIを終了します。

1.6.2. View メニュー

1.6.2.8. Directory View

メインパネルに現在のディレクトリ構成を表示します。パネルのDirectoryタブをクリックすることでも表示できます。

ディレクトリの左のボタンをクリックすることで、ツリーを展開します。ファイルまたはディレクトリをダブルクリックすると、詳細ダイアログが開きます。

詳細ダイアログでは、そのファイルまたはディレクトリが保持しているパーミッションやサイズ、Unique ID情報などを確認することができます。

ダイアログ下部では、イメージ全体においてファイルが使用している領域を赤く表示します。

Saveボタンを押すと、そのファイルまたはディレクトリデータを外部ファイルに書き出します。保存ダイアログが開きますので、ファイル名を指定して保存してください。

Closeボタンを押すと、詳細ダイアログを閉じます。

1.6.2.9. Descriptor View

メインパネルに現在のUDFファイルシステムを、XMLで定義されている構造単位で表示します。パネルのDescriptorタブをクリックすることでも表示できます。

ノードの左側にボタンが表示されている場合、そのノードは子ノードを持っています。ボタンをクリックすることで、ツリーを展開します。各ノードをダブルクリックすると、詳細ダイアログが開きます。

Closeボタンを押すと、詳細ダイアログを閉じます。

1.6.2.10. Verify View

メインパネルンに検証ビューを表示します。パネルのVerifyタブをクリックすることでも表示できます。

Tool メニューのVerifyを選択するか、ツールバーのVerifyをクリックすることで、現在読み込んでいる構造を検証し、その結果を表示します。

1.6.3. Tool メニュー

1.6.3.11. Verify

現在読み込んでいるイメージデータのエラー検証を行います。検証結果は検証ビューに表示されます。

検証結果を表示するには、ツールバーのVerifyをクリックするか、ViewメニューからVerify Viewを選択してください。

1.6.3.12. Option

UDFV GUIの動作環境を設定します。詳細は､1.7項を参照してください。

1.6.4. Help メニュー

バージョン情報を表示します。

1.7. Option ダイアログ

UDFV GUIの動作環境を設定します。

1.7.1. File タブ

Current Directoryで作業用ディレクトリを設定します。ファイルダイアログを開いた際のデフォルトディレクトリになります。

1.7.2. IP タブ

ATAPIServer.exeが動作しているWindows PCのIPアドレスとドライブ文字を設定します。

  • IP Address: ATAPIServer.exeが動作しているWindows PCのIPアドレスを指定します。
  • Drive: メディアが挿入されているドライブのドライブ文字を指定します。

1.7.3. Image タブ

現在読み込んでいるイメージデータが、どのメディアに、どのアクセスタイプで記録されているかを設定します。この情報は、Verifyを行う際に使用されます。

Verifyでは、これらの情報を設定していない場合、DVD-RWにOverwritableで記録されているものとして検証を行います。

現在この機能はサポートされていません。

1.8. Space Bitmap パネル

イメージデータを読み込むと、UDFV GUIウィンドウ下部のSpace Bitmapパネルにビットマップの情報が表示されます。このパネルでは、Space Bitmap Descriptorに記録されている領域情報だけではなく、0セクタからNセクタまでメディア上の全てのビットマップ情報を表示します。(Nは最終セクタ)

また、UDFファイルシステム上にSpace Bitmap Descriptorが記録されている必要はありません。例えば、DVD-ROMやBD-ROMのイメージデータを読み込んでも、ビットマップ情報を表示することができます。

灰色で表示される領域は、UDFファイルシステムが使用していない領域です。ただし、何らかのデータが書き込まれている可能性があります。書かれているデータの内容については保証されていません。

緑色で表示される領域は、ボリューム領域です。ボリューム情報はパーティション外に記録されている情報で、Main Volume Descriptor SequenceやReserve Volume Descriptor Sequence、Anchor Volume Descriptor Pointerなどが含まれます。

青色で表示される領域は、Type1 Partition Map領域です。通常はFile EntryやFile Identifier Descriptor、ファイルのデータ部などが含まれます。

その他の色については、Type1 Partition Map以外のパーティションマップ領域を示しています。通常のイメージであれば、パーティションマップは2つしか存在しません。(Type1 Partition MapがPartition Reference Number 0)

その場合、Partition Reference Numberが1のパーティションマップ領域は黄色で表示されます。主にMetadata Partition Map、Virtual Partition Mapが該当します。

この黄色の領域は、あくまでType1 Partition Map側から見た領域です。例えば、Metadata Partition内にMetadata Bitmapが記録されており、それが空き領域の存在を示していたとしても、その領域が灰色で表示されることはありません。

また、パーティションマップが複数存在する場合、パーティションマップ番号が大きいパーティションマップ領域が上に重ねて表示されます。
